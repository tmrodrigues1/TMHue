# Contribuindo

Obrigado pelo interesse em melhorar o TMHue.

1. Abra uma issue antes de mudanças amplas.
2. Mantenha a arquitetura em camadas e não introduza dependências de rede sem discussão.
3. Execute `dotnet build TMHue.sln -c Release` e `dotnet test TMHue.sln -c Release` antes de enviar uma alteração.
4. Não inclua arquivos de `bin/`, `obj/`, `artifacts/`, certificados ou credenciais no commit.

Ao enviar uma contribuição, descreva o comportamento alterado e os testes realizados.
